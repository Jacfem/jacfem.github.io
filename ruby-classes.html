Classes & Instance Variables

An Introduction

As mentioned in my previous posts, Ruby is an object-oriented programming language. Put simply, nearly every thing we interface with while using Ruby is an object. If you think about all of the different objects in the world, you’ll realize that they each can have very different properties (say, a dog versus a table – one is active, one is static) or very similar properties (say, a table and a chair). Objects in ruby can be given with the same consideration. Do strings behave the same way as integers? Do arrays have the same functionality as hashes?

The answer is of course, no. All of the examples above are objects, but they each behave differently. This is because they are all objects of a different Class.

For starters, if you’d like to play around in IRB and see some various Classes, you can type # .class on any object and it will return its Class.

So, Ruby has built-in Classes, some of which are identified in IRB above. 1 is an instance of a Fixnum Class, “Hi!” is an instance of a String Class, and so on.

But what if we want to define our own Class, and instances of it, and why would we?

Say for example we need to create several methods that will need access to the same variables and information, but do separate things and need to be defined as multiple methods. An example could be creating an Rsvp class that could be useful for keeping track of a guest list for a meetup.

The steps you would take are:

-Define your class

-Define an initialize method which will pass down any parameters defined in this to future instances of the class

This is where instance variables go. Instance variables are preceded by the @ symbol. We create instance variables because each new object of the Rsvp Class we create will be an instance of the Class, and thus have different instances of the variables (aka different values will be stored for these parameters on each new object!)

-End the method, End the Class

-Create a new instance of the Class and pass it the parameters defined in your initialize method

An instance of a Class is an object of that Class. If we type person.class, per our IRB exercise above, it says Rsvp (which is correct!).
